# 礼往来

礼往来是一款轻量的人情礼簿 iOS App，用来记录收礼、送礼、回礼提醒、按人查询往来历史，以及按「我家办的一场事」集中记账。

核心定位：

> 查旧账，回礼更体面。

## 当前已完成

- 首页总览：回礼提醒、年度收送统计、最近入簿、快捷操作。
- 快捷入口：首页支持「收礼」「送礼」「一场事」「查旧账」。
- 一场事：可先新建我家办的婚礼、满月、乔迁等事件，再进入事件详情连续新增收礼记录。
- 礼簿：按类型、时间筛选，支持搜索、编辑、删除记录；已加入「一场事」入口卡片。
- 礼簿详情：支持查看完整字段、编辑记录、删除记录。
- 入簿：支持收礼/送礼、姓名、金额、事件、日期、关系、地点、礼品、联系方式、备注等字段；更多信息默认收起。
- 人情：按人聚合往来历史，支持关系筛选和姓名搜索。
- 人情详情：展示个人往来总览、待回礼状态和历史明细。
- 我的：主题切换、导出 Excel、关于礼往来。
- 导出：已升级为真正的 `.xlsx` 文件，移除 CSV 导出入口。
- 空状态和异常状态：补充了无记录、无搜索结果、无提醒、导出失败等基础提示。
- 视觉：标题使用思源宋体，正文/明细使用楷体；宣纸底、朱砂/墨青/暖金主题、透明通道素材装饰已接入。
- 上架准备：已补 `PrivacyInfo.xcprivacy`、iPhone-only 设备目标和 1024 AppIcon 占位图。
- 数据：`GiftRecord` 和 `HostedGiftEvent` 使用 SwiftData 本地存储；主题使用 UserDefaults 保存；运行时不再注入模拟数据。

## 真实数据持久化

当前 App 已经使用 SwiftData 落盘保存数据，不是纯内存数据。

- 真实礼簿记录模型：`GiftRecord`
- 我家办的一场事模型：`HostedGiftEvent`
- 存储方式：`ModelConfiguration(isStoredInMemoryOnly: false)`
- 主题偏好：`UserDefaults`，key 为 `liwanglai.selectedTheme`
- 模拟数据：已移除运行时 `MockData` 注入，Debug 和 Release 都不会自动生成礼簿记录。
- 真实数据：用户新增的数据会写入 SwiftData，作为真实数据长期保留。

后续在上架或 TestFlight 前，需要重点检查 SwiftData schema 迁移策略。当前新增字段和新增模型较多，如果已经安装过旧版本 App，可能需要补轻量迁移验证或清理旧调试数据。

## 后续优化计划

### 第一优先级

- 补强真实数据持久化：为 SwiftData 增加迁移验证，确认旧版本升级不丢数据。
- 优化一场事数据关系：让 `GiftRecord` 持有 `HostedGiftEvent` 的明确关联 ID，避免同一天同类型事件被误归并。
- 增加完整备份/恢复：导出 App 原生备份包，并支持从备份恢复。
- 增加 Excel 导入：和现有 `.xlsx` 导出形成闭环。
- 准备 App Store Connect 必填项：隐私政策 URL、支持 URL、截图、描述、关键词、分类和年龄分级。

### 第二优先级

- 增加本地通知：把回礼提醒接入系统通知，并提供提醒时间设置。
- 完善一场事编辑：支持修改事件名称、日期、类型、备注，支持删除一场事。
- 强化空状态：针对新用户首页、礼簿、人情、提醒页做更完整的新手引导。
- 增加数据校验：姓名、金额、日期、联系方式等字段增加更明确的输入限制和错误提示。
- iPad 端优化：恢复并打磨横屏记账台，补 iPad 专属布局和截图。

### 第三优先级

- iCloud 同步：在本地持久化稳定后接入 CloudKit。
- Face ID 解锁：提供隐私保护入口，并设计忘记/失败后的回退方式。
- 多主题细化：继续对朱砂、墨青、暖金主题做细节配色和素材适配。
- 字体资产内置：如果系统楷体不够接近原型，后续可内置更明显的楷体字体文件。

## 运行方式

项目是 Xcode iOS 工程，最低系统版本 iOS 17。

```bash
open LiWangLai.xcodeproj
```

也可以直接用命令行构建：

```bash
xcodebuild -project LiWangLai.xcodeproj -scheme LiWangLai -destination 'platform=iOS Simulator,name=iPhone 17' build
```
