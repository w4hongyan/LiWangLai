# 礼往来

礼往来是一款轻量的人情礼簿 iOS App，用来记录收礼、送礼、回礼提醒、按人查询往来历史，以及按「我家办的一场事」集中记账。

核心定位：

> 查旧账，回礼更体面。

## 当前已完成

- 首页总览：回礼提醒、年度收送统计、最近入簿、快捷操作。
- 快捷入口：首页支持「收礼」「送礼」「一场事」「查旧账」。
- 一场事：可新建、编辑和删除我家办的婚礼、满月、乔迁等事件；修改日期或类型会同步关联记录，删除事件会保留礼簿记录并解除归属。
- 一场事双模式：iPad 场次卡片提供「默认模式」和「礼台模式」两个入口；默认模式进入场次详情，礼台模式直接带入当前场次进行现场收礼。
- 礼簿：按类型、时间筛选，支持搜索、编辑、删除记录；已加入「一场事」入口卡片。
- 礼簿详情：支持查看完整字段、编辑记录、删除记录；不使用「已回/未回」概括一段往来，多次收礼和多次送礼均作为独立记录保存。
- 入簿：支持收礼/送礼、姓名、金额、事件、日期、关系、地点、礼品、联系方式、备注等字段；日期同步显示公历和农历，更多信息默认收起。
- 人情：按人聚合往来历史，支持关系筛选和姓名搜索。
- 人情详情：展示个人往来总览、待回礼状态和历史明细。
- iPad 基础适配：工程同时支持 iPhone 和 iPad；两端统一使用「今日、礼簿、入簿、人情、我的」五个底部菜单。
- iPad 横屏礼台：今日页提供礼台入口，顶部使用文字 Logo；未选择或新建一场事前禁止录入，选择后显示该场次的统计和最近记录，并支持常用金额、备注与连续登记。
- iPad 礼台入口：可从今日页进入后选择场次，也可从一场事列表的「礼台模式」按钮或场次详情直接进入并自动选择当前场次；竖屏时显示横屏提示。
- 我的：主题切换、Face ID / Touch ID 解锁、Excel 导入导出、去重合并、系统通知、完整备份与恢复、关于礼往来。
- Excel：支持 `.xlsx` 导入导出，包含公历、农历和提醒时间；导入前展示预览，自动跳过重复项和无效行。
- 去重合并：只识别姓名、日期、事件、方向和金额均一致的明确重复记录，确认后保留信息较完整的一笔。
- 通知：收礼可设置回礼提醒，送礼可设置送礼提醒；用户主动授权后由 iPhone 在本地按日期和时间通知。
- 备份：支持导出 App 原生 JSON 完整备份，并在确认后恢复礼簿和「一场事」数据。
- 空状态和异常状态：补充了无记录、无搜索结果、无提醒、导出失败等基础提示。
- 视觉：标题使用思源宋体，正文/明细使用楷体；宣纸底、朱砂/墨青/暖金主题、透明通道素材装饰已接入。
- 上架准备：已补 `PrivacyInfo.xcprivacy`、iPhone+iPad 设备目标和正式 1024 AppIcon。
- 数据：`GiftRecord` 和 `HostedGiftEvent` 使用带版本声明的 SwiftData 本地存储；主题和解锁偏好使用 UserDefaults 保存；运行时不再注入模拟数据。
- 可靠性：保存、修改、删除失败会回滚并提示；「一场事」使用明确事件 ID 关联；回礼完成会同步清理提醒。
- 容错：数据库无法打开时会进入保护页，不会直接退出或自动覆盖原有礼簿。
- 自动化验证：当前完整测试集为 93 项，覆盖记录、查询、提醒、导入导出、备份和数据服务等基础逻辑。

## 真实数据持久化

当前 App 已经使用 SwiftData 落盘保存数据，不是纯内存数据。

- 真实礼簿记录模型：`GiftRecord`
- 我家办的一场事模型：`HostedGiftEvent`
- 存储方式：`ModelConfiguration(isStoredInMemoryOnly: false)`
- 主题偏好：`UserDefaults`，key 为 `liwanglai.selectedTheme`
- 模拟数据：已移除运行时 `MockData` 注入，Debug 和 Release 都不会自动生成礼簿记录。
- 真实数据：用户新增的数据会写入 SwiftData，作为真实数据长期保留。

项目已建立版本化 SwiftData schema 和迁移计划入口，并补充内存容器测试。上架或 TestFlight 前仍需拿真实旧版本安装包做一次覆盖升级，确认历史数据库可无损打开；这一步不能只靠新安装测试替代。

## 后续优化计划

### 第一优先级

- 使用旧版真实数据库执行覆盖升级测试，确认升级不丢数据。
- 在真实 iPhone 和 iPad 上安装测试，重点检查横竖屏切换、外接/软键盘、连续录入和后台恢复。
- 准备 App Store Connect 必填项：隐私政策 URL、支持 URL、截图、描述、关键词、分类和年龄分级。
- 完成 Archive、TestFlight 内测和正式版本上传流程。

### 第二优先级

- 强化空状态：针对新用户首页、礼簿、人情、提醒页做更完整的新手引导。
- 增加数据校验：姓名、金额、日期、联系方式等字段增加更明确的输入限制和错误提示。
- iPad 端优化：继续整合横屏礼簿管理页面，补齐分屏、多任务、键盘和不同尺寸 iPad 的适配。

### 第三优先级

- iCloud 同步：在本地持久化稳定后接入 CloudKit。
- 礼台留言：先增加手写签名，再评估录音和视频留言及其备份、隐私与空间成本。
- 多主题细化：继续对朱砂、墨青、暖金主题做细节配色和素材适配。
- 字体资产内置：如果系统楷体不够接近原型，后续可内置更明显的楷体字体文件。

## 运行方式

项目是 Xcode iOS 工程，最低系统版本 iOS 17。

```bash
open LiWangLai.xcodeproj
```

也可以直接用命令行构建：

```bash
xcodebuild -project LiWangLai.xcodeproj -scheme LiWangLai -destination 'platform=iOS Simulator,name=iPhone 17' build
```
