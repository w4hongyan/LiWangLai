# 礼往来修复计划

更新时间：2026-06-30

## 当前结论

当前工程还没有达到可运行验收状态。`flutter analyze`、`flutter test`、iOS Simulator 构建均失败，核心阻断是新旧数据模型并存：

- `lib/main.dart` 中定义了一套旧的 `GiftRecord`、`GiftDirection`、`EventTone`、`LedgerTotals`、`ReturnGiftAdvisor`。
- `lib/domain/entities/gift_record.dart` 和 `lib/domain/services/return_gift_advisor.dart` 中也定义了新的领域模型。
- `lib/features/*` 页面用旧 UI 组件包装新 repository，导致 Dart 类型不兼容。

因此，后续修复应先解决编译和架构边界，再做按钮闭环、持久化验收、内购和视觉打磨。

## 修复目标

1. 恢复 Flutter 和 iOS 构建通过。
2. 让首页、礼簿、记一笔、查往来、礼台模式、我的、设置全部可达且按钮闭环。
3. 确认记录、编辑、回礼、礼台速记、撤销、待补全、待回礼都写入 Drift 数据库。
4. 补齐设置、导出、导入恢复、提醒、隐私锁等关键闭环。
5. 明确内购 Pro 的真实方案，避免继续展示空按钮。
6. 完成 iPhone/iPad 布局走查，对照 `原型/` 截图修正视觉和交互。

## 阶段 1：恢复可构建

处理方案：

- 保留 `lib/domain/entities/gift_record.dart` 作为唯一真实 `GiftRecord` 定义。
- 保留 `lib/domain/services/return_gift_advisor.dart` 作为唯一真实回礼建议服务。
- 从 `lib/main.dart` 删除或重命名旧的同名实体、枚举、服务类。
- 让旧 UI 组件统一使用 `core/types.dart` 导出的领域模型。

验收标准：

- `rg "class GiftRecord|enum GiftDirection|enum EventTone|class LedgerTotals|class ReturnGiftAdvisor" lib` 中不再出现重复定义。
- `flutter analyze` 不再出现 `GiftRecord/*1*/` 和 `GiftRecord/*2*/` 类型冲突。

### 1.2 整理 App 入口与测试入口

处理方案：

- `lib/main.dart` 只保留真实启动入口和必要导出，避免混入旧演示状态模型。
- `LiWangLaiApp` 从 `lib/app/app.dart` 明确导入，测试不要再依赖旧 `main.dart` 中不存在的构造器。
- 修复 `test/widget_test.dart` 的 ambiguous import。

验收标准：

- `flutter analyze` 通过。
- `flutter test` 至少能完整加载所有测试文件。
- XcodeBuildMCP `build_run_sim` 能完成构建并启动 App。

## 阶段 2：导航与按钮闭环

优先级：P0

### 2.1 首页快捷入口闭环

当前问题：

- 首页“查往来 / 记一笔 / 礼簿”只写入 `homeTabIndexProvider`，没有真正跳转。
- 当前 `/home` 只渲染首页，没有 shell/tab 容器消费 `homeTabIndexProvider`。

处理方案二选一：

- 推荐：实现 `ShellRoute` 或一个真实的 `HomeShellPage`，用底部导航切换首页、礼簿、记一笔、查往来、我的。
- 简化：首页按钮直接 `context.go('/home/search')`、`context.go('/home/add')`、`context.go('/home/ledger')`。

验收标准：

- 首页四个快捷入口都能进入对应页面。
- 底部导航可在五个一级模块间往返。
- 返回行为符合 iOS 习惯，不出现无法返回或重复压栈。

### 2.2 礼台模式闭环

当前已有：

- 快速保存、清空输入、撤销上一条、最近记录 UI。

需要补齐：

- 保存时显式写入 `entryMode = quickDesk`。
- 保存时显式写入 `completionStatus = partial`。
- 最近记录优先显示本场记录，同时能反映数据库刷新后的数据。
- 编辑、删除、批量补全如果暂不做，UI 不要展示成可点能力。

验收标准：

- 礼台保存后，记录出现在礼簿和待补全。
- 撤销后记录软删除，礼簿中消失。
- 白榜/红榜切换后的事件类型、颜色和记录字段一致。

## 阶段 3：设置、隐私、提醒、导入导出

优先级：P1

### 3.1 合并两套设置页

当前问题：

- `/home/settings` 指向 `lib/features/settings/settings_page.dart`，具备隐私锁和导出。
- `我的`页齿轮打开的是 `lib/main.dart` 中旧 `SettingsPage`，只做本地 `setState`，不会持久化。

处理方案：

- 删除或重命名旧 `main.dart` 设置页。
- `我的`页齿轮统一跳转 `/home/settings`。
- 金额隐藏、白榜隐私、通知开关都写入 `SharedPreferences` 或配置表。

验收标准：

- 从“我的”进入设置，看到的是唯一真实设置页。
- 关闭隐私锁后重启 App 不再要求解锁。
- 通知开关状态重启后保持。

### 3.2 隐私锁生命周期

当前已有：

- `PrivacyService` 支持启动锁、解锁、30 秒 grace period。

需要补齐：

- App 根节点实现 `WidgetsBindingObserver`。
- 后台记录 `lastBackgroundedAt`。
- 前台恢复时调用 `PrivacyService.onResume()` 并触发路由重定向到 `/lock`。

验收标准：

- App 进入后台超过 30 秒后再回来，自动进入锁屏。
- 关闭隐私锁后不触发后台再锁。

### 3.3 提醒真实数据

当前问题：

- 首页“即将到来”使用 `SampleData.reminders`。
- `ReminderRepository` 和 `ReminderScheduler` 已有基础能力，但 UI 未接入。

处理方案：

- 首页读取 `remindersStreamProvider`。
- 添加提醒创建入口，至少在回礼或日期相关记录中能创建提醒。
- 调度本地通知与数据库提醒状态保持一致。

验收标准：

- 新建提醒后首页立即显示。
- 通知权限开启后能调度系统通知。
- 标记完成后首页不再显示。

### 3.4 导出与导入恢复

当前已有：

- JSON、CSV、PDF 导出服务。

需要补齐：

- JSON 备份导入恢复。
- 文件选择、schema 校验、冲突处理。
- 恢复前确认弹窗，避免误覆盖。

验收标准：

- 导出 JSON 后可重新导入。
- 导入后记录数量、字段、待补全、待回礼状态一致。
- 错误格式能给出明确提示。
